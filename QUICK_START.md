## 🏃 Quick Start (5 minutes)

### Step 1: Wait for Dependencies
npm install is running... (should complete in 2-3 minutes)

```
Progress: Installing 150+ packages...
⏳ Almost done...
```

Check terminal output below for "added XXX packages" when done.

### Step 2: Create Supabase Project (2 minutes)
1. Go to https://supabase.com
2. Click "Start your project"
3. Sign up and create new project
4. Choose region: Europe (Southeast) Oslo
5. Go to **Settings → API**
6. Copy your **Project URL** and **Anon Public Key**

### Step 3: Configure Environment (1 minute)
1. In TimeplanEz folder, create `.env.local`
2. Add (replace YOUR VALUES):
```
VITE_SUPABASE_URL=https://your-project-id.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key-here
```

### Step 4: Setup Database (1 minute)
1. In Supabase, go to **SQL Editor**
2. Click **New Query**
3. Paste entire content from: `supabase/migrations/001_init.sql`
4. Click **Run** (blue button)
5. Wait for success ✅

### Step 5: Run Locally (1 minute)
```bash
npm run dev
```
Your browser opens → Sign up → Start logging hours!

## 💡 Key Concepts

### Paid vs Unpaid Logic
```
Scenario: You set 20 school hours/week

Monday:  8 hours logged → Marked as UNPAID (total unpaid: 8)
Tuesday: 8 hours logged → Marked as UNPAID (total unpaid: 16)
Wednesday: 8 hours logged → Marked as PAID (total unpaid would be 24, exceeds 20!)
```

Result:
- 🔴 **Unpaid: 16 hours** (school hours)
- 🟢 **Paid: 8 hours** (counted in earnings)
- 💰 **Earnings: 8h × €150/h = €1,200**

You can also manually override - mark any shift as paid/unpaid.

- **No signup forms** - Just email/password

## 🆘 Troubleshooting

### "npm install is taking forever"
Normal! With 150+ packages, takes 2-3 minutes. Be patient. ☕

### "Cannot connect to Supabase"
- Check .env.local has correct URL and key
- Verify Supabase project is active
- Try logging out and back in

### "GitHub Pages shows blank"
- Wait 2-3 minutes for build
- Check Actions tab for errors
- Verify Settings → Pages shows "/dist"

### "Button doesn't work"
- Open browser console (F12)
- Look for error messages
- Check .env.local is set correctly
